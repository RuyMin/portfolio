## Overview

- 상태: 운영 중
- 기술: Python, FastAPI, CrewAI (GPT-4o-mini), Apache Kafka, PostgreSQL, Docker Compose, gRPC, Grafana/Loki, GitHub Actions
- 인프라: Oracle Cloud Free Tier (ARM Ampere A1)
- GitHub: https://github.com/RuyMin/stock-signal_v2

## Why I Built This

실무에서 다뤄온 비동기 파이프라인, Kafka 메시지 기반 아키텍처, AI 연동 경험을 개인 프로젝트로 검증하고 싶었습니다. 동시에 실제로 매일 사용할 수 있는 종목 추천 도구를 만들어 실사용 피드백 루프를 돌리는 것이 목표였습니다.

## Architecture

전체 시스템은 12개 Docker 컨테이너로 구성된 이벤트 드리븐 아키텍처입니다.

1. **Scheduler** — APScheduler 기반 2-cron (장중 16:30 수급 수집 / 장전 06:30 매크로+뉴스+추천)
2. **Worker (Data Collector)** — 한국투자증권 OpenAPI로 기관/외국인 수급 수집, yfinance로 미국 매크로 5지표, 네이버 뉴스 스크래핑
3. **CrewAI (AI Orchestration)** — 4개 Agent (Signal/News/Macro/Synthesizer)가 순차 분석 후 스팸 스코어 산정
4. **Worker (Telegram Notifier)** — 추천 결과를 multi-user fan-out으로 텔레그램 전송
5. **Worker (Telegram Listener)** — 사용자 명령어 처리 (보유 종목 관리, 추천 조회 등)
6. **Backend (FastAPI)** — REST API, PostgreSQL CRUD, 헬스체크
7. **Infra** — Kafka + Zookeeper, PostgreSQL, Loki + Promtail + Grafana

## Key Features

- 기관/외국인 3일 이상 연속 순매수 종목 자동 탐지
- GPT-4o-mini 기반 수급·뉴스·매크로 종합 점수 산정 (가중치 50/25/25)
- 보유 종목 탈출 타이밍 알림 (exit_alert)
- 텔레그램 봇 인터페이스 (10개 명령어)
- Multi-user 화이트리스트 + admin 승인 체계
- KRX 휴장일 자동 스킵 + 휴장일 갭 뉴스 범위 확장
- Grafana 대시보드로 job_id 기반 전체 파이프라인 추적
- CI/CD: GitHub Actions (테스트 → 6서비스 빌드 → GHCR push → SSH 배포)

## Technical Decisions

- **Kafka 비동기 파이프라인**: 모듈 간 결합도를 낮추고, 장애 격리와 재처리를 용이하게 하기 위해 선택
- **CrewAI + GPT-4o-mini**: LLM 오케스트레이션 프레임워크로 Agent 역할 분리. 비용 효율을 위해 mini 모델 사용
- **Docker Compose 단일 VM**: Oracle Cloud Free Tier에서 12컨테이너를 ~3GB 메모리로 운영
- **코드 기반 분류 강제**: LLM의 비결정적 출력을 on_complete()에서 점수 기반으로 재분류하여 일관성 확보
- **structlog + job_id 바인딩**: 전체 파이프라인을 단일 ID로 추적 가능한 관측 환경

## What I Learned

- Kafka 기반 이벤트 드리븐 아키텍처의 실제 운영 경험 (poison message 처리, 직렬화 일원화)
- CrewAI/LLM 출력의 비결정성을 코드로 보정하는 패턴
- 한국투자증권 OpenAPI 실전 연동 (OAuth 토큰 캐시, TR_ID 관리)
- Docker Compose 멀티 서비스 환경에서의 CI/CD 파이프라인 구축
- 단위 테스트 129건 컨테이너 격리 실행 전략
