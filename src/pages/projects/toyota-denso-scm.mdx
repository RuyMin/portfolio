## Overview

- 기간: 2016.06 - 2020.11
- 역할: 선임 개발자 / 시스템 아키텍처 설계
- 도메인: 자동차 제조/물류
- 대상 지역: 일본 (현지 근무)
- 기술: C#, Java, OracleDB, SCM 데이터 모델링

## Context

일본 현지에서 토요타/덴소의 글로벌 자동차 부품설계 및 물류 프로세스를 시스템으로 전환하는 프로젝트에 참여했습니다. 기존에 엑셀 기반 수작업으로 관리되던 업무를 요구사항 정의부터 데이터 모델링, 화면 설계, 코어 백엔드 개발까지 폭넓게 수행했습니다.

글로벌 물류 인프라 표준 가이드를 수립하고, 재사용 가능한 데이터 구조와 업무 흐름을 정리했습니다.

## My Contributions

- 요구사항 정의에 참여하여 수작업 업무를 시스템 요구사항으로 전환했습니다.
- SCM 데이터 모델링을 수행했습니다.
- 화면 설계 및 코어 백엔드를 개발했습니다.
- 글로벌 물류 인프라 표준 가이드를 수립했습니다.

## Problems

### 수작업 프로세스의 시스템화
엑셀 기반으로 관리되던 부품설계 및 물류 프로세스를 시스템으로 전환하면서, 암묵적인 업무 규칙을 명시적인 요구사항으로 정리해야 했습니다.

### 글로벌 물류 데이터 구조 설계
여러 국가/거점에서 재사용 가능한 데이터 구조를 설계해야 했고, 각 거점별 업무 차이를 공통 모델 안에서 수용해야 했습니다.

### 일본 현지 협업 환경
일본어 기반 요구사항 정의, 현지 고객과의 직접 커뮤니케이션, 일본 개발 문화에 맞춘 협업 방식이 필요했습니다.

## Technical Points

- 엑셀 수작업 업무를 시스템 요구사항으로 전환하는 분석 방법
- SCM 도메인 데이터 모델링 및 글로벌 거점 대응 구조 설계
- C#/Java 기반 코어 백엔드 개발
- 글로벌 물류 표준 가이드 수립

## Impact

- 수작업 중심 업무를 시스템 기반 프로세스로 전환했습니다.
- 글로벌 제조/물류 환경에서 재사용 가능한 데이터 구조와 업무 흐름을 정리했습니다.
- 4년간 일본 현지에서 요구사항 정의부터 운영까지 전 과정을 경험했습니다.
