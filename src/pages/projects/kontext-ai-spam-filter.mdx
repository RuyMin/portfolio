## Overview

- 기간: 2020.12 - 2022.07
- 역할: 백엔드 개발 핵심 멤버
- 도메인: 모바일 AI 서비스
- 기술: Java, Spring Boot, gRPC, Apache Kafka, REST API, Docker, MSA, ELK

## Context

KONTEXT는 현재 안드로이드/아이폰에서 기본 제공하는 통화 스크린 기능과 유사한 서비스입니다. 수신 통화 시 상대방의 음성을 실시간으로 분석하여 스팸 여부를 판단하고 사용자에게 알려줍니다.

전체 아키텍처는 다음과 같습니다:
1. 모바일 앱에서 상대방 음성 데이터를 gRPC로 실시간 전송
2. gRPC로 수신한 메시지를 Kafka에 적재
3. 백엔드(Spring Boot)에서 Kafka 메시지를 소비
4. RESTful API로 미국 본사의 AI 플러그인에 전달하여 스팸 스코어를 수신
5. 스코어 기반으로 스팸 여부를 판단하여 사용자에게 결과 전달

## My Contributions

- Kafka Consumer 기반 메시지 처리 파이프라인을 구현했습니다.
- 미국 본사 AI 플러그인과의 RESTful 연동 인터페이스를 개발했습니다.
- 스팸 스코어 판정 로직 및 사용자 응답 처리를 구현했습니다.
- 대량 동시 사용자 대응을 위한 서버 아키텍처를 개선했습니다.
- ELK 기반 로그 수집 및 운영 관측 환경을 구축했습니다.

## Problems

### 대량 동시 사용자 대응
다수의 사용자가 동시에 통화할 때 gRPC로 유입되는 실시간 메시지를 안정적으로 처리해야 했습니다. Kafka를 버퍼로 활용하여 백엔드의 처리 부하를 분산했습니다.

### AI 플러그인 연동 안정성
미국 본사의 AI 플러그인에 RESTful로 스코어를 요청할 때, 네트워크 지연이나 장애 상황에서도 서비스가 중단되지 않도록 장애 격리와 재시도 전략을 설계해야 했습니다.

### 실시간 처리 요구사항
상대방의 음성이 실시간으로 들어오는 구조이기 때문에, Kafka 소비부터 AI 스코어 응답까지의 전체 레이턴시를 최소화해야 했습니다.

## Technical Points

- gRPC 수신 → Kafka 적재 → Spring Boot Consumer → REST AI 연동의 비동기 파이프라인 구조
- Kafka 기반 메시지 버퍼링으로 트래픽 급증 대응
- 외부 AI API 연동 시 타임아웃/재시도/장애 격리 설계
- ELK 기반 실시간 로그 수집 및 운영 관측 환경 구축

## Impact

- 대량 동시 사용자 환경에서 안정적인 실시간 스팸 판정 파이프라인을 구축했습니다.
- AI 플러그인 연동 구간의 장애 격리를 통해 서비스 안정성을 확보했습니다.
- ELK 기반 관측 환경으로 운영 중 장애 감지 및 대응 속도를 높였습니다.
